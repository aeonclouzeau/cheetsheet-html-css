Web Fonts

A web font is any font used in a website's design that isn't installed by default on the end user's deviceâ€”a counterpart to a system font.

Use web fonts in your website utilizing one of these three formats:

Format Description
<link /> When it comes to web fonts, the link element is often used to connect to font files hosted on external servers or content delivery networks.
@import Utilized to import font files from external sources or to include font-face declarations.
@font-face Allows to specify the font files to be downloaded and used on a website, even if those fonts are not installed on the user's system.

Link

When selecting a font and its styles, you can review your selected font family, and a <link/> element will be automatically generated.

Apply web font via <link/> 

							
	<head>
	  <link href="https://fonts.googleapis.com/css2?family=Roboto:wght@100&display=swap" rel="stylesheet" />
	</head>

The generated <link/> element is added inside the scope of the <head> element in the HTML document for it to be ready to be used in the CSS stylesheet. 


								
				p { font-family: 'Roboto', sans-serif; }

Note: Remember to always use fallback fonts in your font stack. In this example, the fallback font is sans-serif, since the Roboto font belongs to the sans-serif font family.

@import

The @import at-rule , as the name implies, is used to import stylesheets and font files from external sources or to include font-face declarations.

Apply web font via @import 

							
	@import url(//fonts.googleapis.com/css?family=Open+Sans);

The generated @import at-rule uses the url() method to import the font Open Sans from Google fonts.

Now you can use the Open Sans font-family in your CSS stylesheet.

								
	p { font-family: 'Open Sans', sans-serif; }

@font-face

The @font-face at-rule sets the sources to the relative path to the font's source, rather locally or .

Apply web font via @font-face 

							
        @font-face {
         font-family: 'MyWebFont';
         src: url('fonts/webfont.eot'); /* IE9 Compat Modes */
         src: url('fonts/webfont.eot?#iefix') format('lembedded-opentype'), /* IE6-IE8 */
              url('fonts/webfont.woff2') format('woff2'), /* Super Modern Browsers */
              url('fonts/webfont.woff') format('woff'), /* Pretty Modern Browsers */
              url('fonts/webfont.ttf') format('truetype'), /* Safari, Android, iOS */
              url('fonts/webfont.svg#svgFontName') format('svg'); /* Legacy iOS */
								  }

Here's what the code example above does:

  • @font-face at-rule is used as the selector.
  • font-family property is used to set a custom name for the downloaded font.
  • src contains the url and the format for each font font format.
  • url() contains the relative path or rather the link to the font file. In this example the font files are stored inside a folder named fonts/ within the working directory.
  • format() tells the web browser what font type the file is.

Once the @font-face at-rule is defined, refer to the custom font as any other in the stylesheet. 

								
	p { font-family: 'MyWebFont', sans-serif; }
Note

The ordering of the different formats is important because the browser will start from the top of the list and search until it finds a font format that it supports.

Font formats

Fonts can be downloaded just like any other file on the web. They come in a few different file formats, such as:

  • TTF (TrueType Font)
  • OTF (OpenType Font)
  • WOFF (Web Open Font Format)
  • WOFF2 (Web Open Font Format 2)
Note

The different formats are a progression of standards for how fonts will work with different browsers, with WOFF2 being the most progressive. It's a good idea to include TTF, WOFF, and WOFF2 formats with your @font-face rule to ensure compatibility on all browsers.

Consider web safe fonts for small screens

Many small devices may have crappy connections. One trick might be to target larger screens when loading the custom font using @media

Tip 
								
        @media (min-width: 1000px) {
          body {
           font-family: 'FontName', Fallback, sans-serif;
          }
        }

In this example, screens smaller than 1000px will render with a web safe font instead and screens that wide and above will be served the custom font.
